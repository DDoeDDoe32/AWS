A라는 모 중학교 선생님이 매번 반복되는 서류 처리를 DB화 시키려고 한다.
-> 중학교, 선생님, 서류, DB

선생님께서 전산화하려고 하는 업무는 학년이 끝날 때 어느 반의 어떤 과목이 규정된 수업시간과 맞는지를 비교하려고 하는 것이다.
-> 학년, 반, 과목, 수업시간
=> 학년, 반, 과목

학년별로 해당 과목의 규정 수업시간은 정해져 있으며 학년 말에 과목별 실제 수업일수가 규정된 수업시간과 맞는지를 체크하는 부족한 경우에는 보강 등을 통해서 규정된 수업시간을 채워야 하므로 이 데이터는 정확해야 한다.
-> 학년, 과목, 규정 수업시간, 수업일수

현재 학년별 각 반의 수업일수를 계산하기 위해서 각 반의 수업일자를 과목별로 일일이 수작업으로 확인하고 있다.
-> 학년 수업, 반, 수업일자, 과목

학년, 반, 과목
학년 - 반
학년 반이 소속되어진다
학년 - 과목
반에 수업 되어진다.

1. 엔터티 도출하시오.
학년, 반, 과목

2. 속성 도출하시오.
학년 : 학년코드(O)
반    : 반 코드(O)
과목 : 과목코드(O), 과목명

3. 식별자 도출하시오.
4. 관계를 도출하시오.
학년 - 소속 - 반
각 학년에 "하나 이상의" 반이 "소속되어질지도 모른다."
각 반은 반드시 "하나의" 학년에 "소속되어질지도 모른다."

학년 - 과목 - 개설
각 반에는 "하나의" 과목이 '수업 되어질지도 모른다."
각 과목은 "하나 이상의" 반에서 "수업이 되어질지도 모른다."

학년:반    -> 1:M => 소속
학년:과목 -> M:N => 개설(규정 수업시간)
반   :과목 -> M:N => 수업(수업시간, 수업일자)

5. 논리적 데이터베이스 모델링 정의하시오.
